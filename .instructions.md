## BACKEND AUTO-DETECTION LOCK — SYMBOL / TIMEFRAME

### CONSTAT VALIDÉ
- La détection du symbole et de la timeframe est dynamique côté TradingView / extension
- Le blocage vient du backend quand il filtre avec des listes fixes

### RÈGLE ABSOLUE
Le backend ne doit JAMAIS bloquer un symbole ou une timeframe reçus depuis TradingView avec une liste fermée.

INTERDICTIONS ABSOLUES :
- VALID_SYMBOLS bloquant
- VALID_TIMEFRAMES bloquantes
- hardcode métier de symbole
- hardcode métier de timeframe
- fallback fixe de type XAUUSD / BTCUSD / EURUSD
- fallback fixe de type M1 / M5 / M15 / H1

### OBLIGATION BACKEND
server.js doit :
- accepter tout symbole reçu depuis TradingView
- accepter toute timeframe reçue depuis TradingView
- stocker la valeur réelle
- propager la valeur réelle
- ne jamais remplacer la valeur réelle par une valeur fixe

- la détection dynamique TradingView reste prioritaire
- un mapping comme SYMBOL_MAP ne peut servir qu’à normaliser l’affichage
- ce mapping ne doit jamais bloquer la donnée
- ce mapping ne doit jamais remplacer la valeur réelle si TradingView a déjà fourni une valeur valide

- sync timeframe : OUI/NON
- bloc exact modifié : ...
